# Render.com Deployment Guide

## Why Render.com for Backend?

✅ **Better for NestJS** than Vercel serverless:
- No cold starts (always-on instance)
- Proper MongoDB connection pooling
- Traditional Node.js runtime (not serverless)
- Free tier includes 750 hours/month (enough for 24/7)
- Auto-sleep after 15 min inactivity (free tier) - wakes up on request

✅ **Features**:
- Automatic HTTPS
- Auto-deploy from GitHub
- Free PostgreSQL/Redis (if needed later)
- Built-in health checks
- Environment variable management

## Step-by-Step Deployment

### 1. **Push Backend Code to GitHub**

```bash
cd house-design-platform
git add backend/render.yaml
git commit -m "Add Render deployment config"
git push origin main
```

### 2. **Sign Up / Log In to Render**

1. Go to https://render.com
2. Sign up with GitHub account (recommended for auto-deploy)
3. Authorize Render to access your repository

### 3. **Create New Web Service**

1. Click **"New +"** → **"Web Service"**
2. Connect your GitHub repository: `20SB/plot2plan`
3. Render will auto-detect `render.yaml`
4. **Important**: Select **"Use render.yaml"** option

### 4. **Configure Environment Variables**

Render will create the service from `render.yaml`, but you need to add the **MongoDB URI** manually:

1. Go to service **"Environment"** tab
2. Add environment variable:
   ```
   Key: MONGODB_URI
   Value: mongodb+srv://sbdbu:bvhk3kJSgnTnelyy@sb.calnbgq.mongodb.net/plot2plan
   ```
3. Click **"Save Changes"**

**Note**: `JWT_SECRET` is auto-generated by Render. Other variables are set in `render.yaml`.

### 5. **Deploy**

1. Render will automatically start the first deployment
2. Wait 3-5 minutes for build and deployment
3. Watch the logs in real-time

### 6. **Get Backend URL**

Once deployed, you'll get a URL like:
```
https://plot2plan-backend.onrender.com
```

### 7. **Update Frontend Environment Variable**

Update the frontend to point to your new backend:

```bash
cd house-design-platform/frontend
```

Edit `.env.local` (create if doesn't exist):
```env
NEXT_PUBLIC_API_URL=https://plot2plan-backend.onrender.com/api/v1
```

Commit and push:
```bash
git add .env.local
git commit -m "Update API URL to Render backend"
git push origin main
```

Vercel will auto-deploy the updated frontend.

### 8. **Verify Deployment**

Test your backend:
```bash
curl https://plot2plan-backend.onrender.com/api/v1
```

Should return: `{"message": "Plot2Plan API is running"}`

Test registration:
```bash
curl -X POST https://plot2plan-backend.onrender.com/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "Test123!@#",
    "name": "Test User"
  }'
```

## Environment Variables Reference

| Variable | Value | Source |
|----------|-------|--------|
| `NODE_ENV` | `production` | render.yaml |
| `PORT` | `10000` | render.yaml (Render default) |
| `MONGODB_URI` | `mongodb+srv://...` | **Manual entry in Render dashboard** |
| `JWT_SECRET` | Auto-generated | render.yaml (generateValue: true) |
| `CORS_ORIGINS` | `https://plot2plan.vercel.app,http://localhost:3000` | render.yaml |
| `API_PREFIX` | `api/v1` | Default in code |

## Auto-Deploy from GitHub

After initial setup, every push to `main` branch triggers auto-deploy on Render.

To disable:
1. Go to service **"Settings"** tab
2. Toggle **"Auto-Deploy"** off

## Troubleshooting

### Issue: "Cannot connect to MongoDB"
**Fix**: Check `MONGODB_URI` in Render environment variables. Ensure:
- Username: `sbdbu`
- Password: `bvhk3kJSgnTnelyy`
- Cluster: `sb.calnbgq.mongodb.net`
- Database: `plot2plan`

### Issue: "Port already in use"
**Fix**: Render automatically assigns `PORT=10000`. Don't override it.

### Issue: "CORS error from frontend"
**Fix**: Add your frontend domain to `CORS_ORIGINS`:
```
CORS_ORIGINS=https://plot2plan.vercel.app,https://your-custom-domain.com
```

### Issue: "Health check failed"
**Fix**: Ensure your backend responds to `GET /` or `GET /api/v1`. 

Check `src/app.controller.ts`:
```typescript
@Get()
getHealth() {
  return { message: 'Plot2Plan API is running' };
}
```

## Manual Deployment (Without render.yaml)

If you prefer manual setup:

1. Create new Web Service
2. Select repository: `20SB/plot2plan`
3. Set **Root Directory**: `backend`
4. Set **Build Command**: `npm install && npm run build`
5. Set **Start Command**: `npm run start:prod`
6. Add environment variables manually (see table above)

## Logs & Monitoring

View logs in real-time:
1. Go to Render dashboard → Your service
2. Click **"Logs"** tab
3. See startup logs, errors, requests

Download logs:
```bash
# Install Render CLI
npm install -g render-cli

# Login
render login

# View logs
render logs plot2plan-backend --tail
```

## Health Check Configuration

Render automatically pings `/` every 60 seconds.

If your app uses a different health check endpoint:

Update `render.yaml`:
```yaml
healthCheckPath: /api/v1/health
```

Add health endpoint in backend:
```typescript
// src/app.controller.ts
@Get('health')
healthCheck() {
  return { status: 'ok', timestamp: new Date().toISOString() };
}
```

## Rollback Deployment

If something breaks:
1. Go to Render dashboard → Service → **"Events"** tab
2. Find previous successful deployment
3. Click **"Rollback"**

Or redeploy a specific commit:
1. Go to **"Manual Deploy"**
2. Select a previous commit hash
3. Click **"Deploy"**
